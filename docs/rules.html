Rules

7 standards applied automatically on every task — no command needed. They are the guardrails behind the workflow.

🔍

codebase-first

Always explore the existing project before generating any code. Prevents architecture conflicts, duplicate implementations, and style inconsistencies.

What it enforces

  • Read project structure, conventions, and dependencies first
  • Reuse existing patterns instead of introducing new ones
  • Surface conflicts before writing code
Source ↗
📋

requirements-first

Before implementing any feature, REQUIREMENTS.md must exist and be consulted — the single source of truth for what to build.

What it enforces

  • No feature work without documented requirements
  • Every task references an F-x / UI-x requirement
  • Scope changes flow through update-requirement
Source ↗
🪶

simplicity-first

Write the minimum code that solves the problem. Nothing speculative. If 200 lines could be 50, rewrite it.

What it enforces

  • No speculative abstractions or premature generalization
  • Prefer the smallest correct solution
  • Refactor toward less code when possible
Source ↗
📐

platform-conventions

Always follow platform-specific conventions for iOS (Apple HIG) and Android (Material Design 3). Detect the target platform and apply the right guidelines.

What it enforces

  • Apple Human Interface Guidelines on iOS
  • Material Design 3 on Android
  • Platform-adaptive components and navigation
Source ↗
🔐

mobile-security

Security best practices applied throughout: data protection, secure storage, network security, and OWASP mobile alignment.

What it enforces

  • Sensitive data in Keychain/Keystore, never plaintext
  • Secure transport and certificate handling
  • OWASP Mobile Top 10 awareness
Source ↗
🚀

performance-standards

Performance budgets and thresholds — launch performance, runtime smoothness, and memory usage.

What it enforces

  • Cold-start and launch performance targets
  • 60fps rendering and scroll budgets
  • Memory and resource ceilings
Source ↗
✅

quality-gates

Pre-commit, pre-PR, and pre-release quality checks. Encodes the blocking-gate doctrine that the orchestrated loop enforces.

What it enforces

  • Checks at commit, PR, and release boundaries
  • Blocking-gate doctrine (intent-level, universal)
  • No debug artifacts or unaddressed TODOs in critical paths
